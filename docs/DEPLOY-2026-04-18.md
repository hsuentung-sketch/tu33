## 2. Render Build Command（選配 — 加強備份品質）

目前 Build Command：
```
npm ci --include=dev && npx prisma generate && npm run fonts:download && npm run build
```

**若要使用 `pg_dump` 做完整 SQL 備份（推薦）**，改為：
```
apt-get update && apt-get install -y --no-install-recommends postgresql-client && npm ci --include=dev && npx prisma generate && npm run fonts:download && npm run build
```

> Render 的 runtime container 沿用 build 階段安裝的系統套件，所以 `pg_dump` 會留在 runtime 中可用。

**若不改 Build Command**：程式會 fallback 用 Prisma JSON 匯出 `.json.gz`，仍可備份 row 內容，但不含 schema DDL（還原時需自行跑 prisma migrate 再寫入）。

## 4. 驗收

### 4a. 系統版本
打開後台任一頁 → 左下角應看到 `v1.0.0 · <短 commit> · <部署時間>`

### 4b. AP 月結對帳單
後台「應付帳款」→ 底部「📄 產生月結對帳單」→ 選供應商 + 年 + 月 → 新分頁開 PDF

### 4c. 備份 — 手動驗證（**部署後先做一次**）
1. 登入後台（ADMIN）取得 session cookie
2. 用 curl / Postman：
   ```
   POST https://erp-line-bot.onrender.com/api/statements/backup
   Cookie: ep_session=<你的 session cookie>
   ```
3. 回應應類似：
   ```json
   { "ok": true, "mode": "sql-gz", "size": 123456, "sentTo": "admin@company.com" }
   ```
4. 到收件信箱確認收到 `backup-YYYYMMDD.sql.gz`（或 `.json.gz` fallback）
5. 若 `mode` 是 `json-gz` → pg_dump 沒裝，看第 2 節決定要不要改 build command

### 4d. 排程
- 每日 **02:00 Asia/Taipei** 自動執行
- 若漏寄，檢查後台「異常紀錄」篩 `source=job.daily-backup`

## 5. 還原程序（災後用）

### SQL 版本（`.sql.gz`）
```bash
gunzip backup-20260418.sql.gz
# 建立空的 database，然後：
psql "<NEW_DATABASE_URL>" < backup-20260418.sql
```

### JSON 版本（`.json.gz`）
需先 `npx prisma migrate deploy` 建好 schema，再用自訂腳本把 JSON 寫回。未來要做一個 `src/tools/restore-json-dump.ts`。
