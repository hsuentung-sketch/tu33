## 快速開始

### 誰適合用哪個介面？

| 情境 | 介面 | 說明 |
|------|------|------|
| 業務開銷貨單、查庫存 | LINE chat | 最快，手機隨時用 |
| 業務開報價單（多品項） | LIFF 報價表單 | 表單介面好輸入 |
| 會計查帳款、關單 | 後台 | 大量資料查閱方便 |
| 管理員建員工、看報表 | 後台 | 必須用後台 |

### 第一次使用要做什麼？

1. **員工綁定 LINE**：管理員幫你產綁定碼，你在 LINE 對話中輸入綁定碼完成
2. **設後台密碼**：請管理員用 CLI 幫你設初始密碼，之後自己改
3. **加入 LINE Bot 為好友**：掃管理員提供的 QR code → 底部出現 Rich Menu

## LINE 前端操作

### Rich Menu（底部圖文選單）

共六格：

| 位置 | 功能 | 角色限制 |
|------|------|----------|
| 左上 | 報價 | 跳 LIFF 報價單 |
| 中上 | 銷貨 | SALES+ |
| 右上 | 進貨 | PURCHASING+ |
| 左下 | 帳務 | ACCOUNTING+ |
| 中下 | 查詢 | 所有人 |
| 右下 | 報價追蹤 | 所有人 |

### 銷貨流程

1. 點「銷貨」或輸入 `銷貨` → 系統問客戶
2. 輸入客戶關鍵字（例：`潤樋`）→ 回 Flex 卡片清單 → 點「選擇」
3. 輸入品項，三種方式：
   - **完整輸入**：`EK-SS-6336 100 21` → 品名 數量 單價
   - **部分輸入 + 搜尋**：輸入 `6336` → 回產品卡片輪播 → 點「選擇」→ 系統接著問數量
   - **僅數量**：用建議售價時只需輸入數量
4. 每加完一項會問：`📝 加備註 / ✅ 直接加入 / ✖ 取消`
5. 品項全加完輸入 `完成` → 系統問**送貨備註**（可輸入文字或 `無`）
6. 確認 → 自動產生銷貨單 + PDF 連結 + 應收帳款

### 進貨流程

與銷貨幾乎相同，差別：
- 對象改成「供應商」
- 單價用「進價」
- 自動產生應付帳款

### 查詢功能

點「查詢」後選類別：

- **產品**：
  - 輸入關鍵字 → 回模糊搜尋結果（Flex 卡片，顯示售價 / 上次成交價 / 交易日）
  - 輸入 `全部` → 列出所有產品名單（Flex bubble 分頁，每頁 25 個按鈕）→ 點選才看詳細 + PDS/SDS/DM 下載
- **客戶**：同上
- **供應商**：同上

### 帳務功能（ACCOUNTING+）

點「帳務」後：
- **應收查詢**：輸入客戶關鍵字 → 未結帳款清單
- **應付查詢**：輸入供應商關鍵字 → 未結應付清單
- **逾期清單**：直接顯示所有逾期未結的帳款

### 名片 OCR（⚙ 僅 ADMIN）

輸入 `管理 → 新增客戶 → 上傳名片照片`：
- 系統用 Google Vision 自動擷取：公司 / 聯絡人 / 電話 / Email / 統編 / 地址
- 若 OCR 不完整，系統會回傳讓你補
- 備援：輸入 `新增客戶 公司/聯絡人/電話/Email/統編/地址`（用 `/` 分隔）

### LINE 綁定碼（⚙ 僅 ADMIN）

- `綁定碼 list` — 顯示未綁定的員工
- `綁定碼 <員工編號>` — 產生 6 位綁定碼給該員工
- 員工在 LINE 輸入那 6 位數完成綁定

## LIFF 報價單

點 Rich Menu「報價」→ 在 LINE 內開啟 LIFF 表單。

### 欄位

- **業務**：自動帶入登入員工（無法改）
- **客戶**：關鍵字搜尋 autocomplete（打兩個字就出現建議）
- **品項（可多筆）**：
  - 品名：關鍵字自動完成
  - 數量 / 單價：自動帶出最新售價，可覆寫
- **備註**
- **有效天數**：預設 30 天

### 送出後

- 自動產生報價單 PDF
- LINE 對話收到「📄 下載 PDF：https://...」短連結
- 若要轉為銷貨單：在 LINE 輸入 `轉銷貨 <報價編號>`

### 主檔管理（客戶 / 產品 / 供應商 / 員工）

- **搜尋**：上方輸入框即時過濾
- **新增**：右上「＋ 新增」→ 填表 → 送出
- **編輯**：列表每列「編輯」按鈕
- **停用**：軟刪除（資料保留在 DB，但列表不顯示），可切到「顯示停用」還原
- **員工頁特殊**：
  - 未綁定 LINE 的員工有「綁定碼」按鈕，點了會產生 6 位數傳給員工
  - **後台登入密碼**（僅 ADMIN）：列表顯示「已設定 / 未設定」狀態；於編輯/新增 modal 中可設定、重設或移除密碼
    - 最少 8 碼；需二次確認；明文密碼不會顯示也不寫 log（DB 存 bcrypt hash）
    - 「移除密碼」後該員工無法登入後台，但 LINE 綁定仍有效

### 產品文件管理

產品列表每列有「文件」按鈕：
- 上傳 PDS / SDS / DM（PDF）
- LINE 端查詢該產品時會自動出現下載按鈕
- 檔案存 Supabase Storage

### 單據檢視（報價 / 銷貨 / 進貨）

- 列表每列「檢視」按鈕 → 彈窗顯示：
  - 單頭（日期 / 客戶 / 業務 / 單號）
  - 品項明細
  - 小計 / 稅 / 總計
  - 送貨備註
  - 「📄 下載 PDF」按鈕

- **編輯**：列表每列「編輯」按鈕（僅 ADMIN 或建單人），已入帳 AR/AP 鎖定
- **刪除**：列表每列紅色「刪除」按鈕（僅 ADMIN 或建單人）
  - 軟刪除：資料保留於 DB，列表立即消失
  - 連動沖銷：銷貨 → 刪除 AR + 庫存回補；進貨 → 刪除 AP + 庫存扣回
  - 已結案（`isPaid=true`）的 AR/AP 無法刪除，需先反結案

> **建單仍在 LINE / LIFF 進行**。後台目前只能查不能建。

### 應收 / 應付帳款

- 列表顯示：單號 / 客戶 / 開票日 / 到期日 / 金額 / 狀態 / 發票號碼
- 到期前 15 天標示黃色，逾期紅色
- 每列「編輯」：標記已付款 + 填發票號碼 + 付款日
- 每列「開立發票」（⚙ ADMIN，應收帳款才有）：帶入客戶資料與銷貨品項 → 取號並產生電子發票 XML
- 底部「📄 產生月結對帳單」→ 選客戶 + 年月 → 自動產生月結 PDF（含未付款月份彙總）

### 會計 ▸ 傳票（Journal Entry）

ACCOUNTING / ADMIN 可進入。所有自動分錄（銷貨/進貨/快速費用/零用金）與手動分錄都會在這裡列出。

**狀態與動作**：
| 狀態 | 可做的動作 |
|------|-----------|
| pending（待審核） | 過帳、**編輯**（v2.7.8+）、刪除（⚙ ADMIN） |
| posted（已過帳）  | 反沖（⚙ ADMIN，產生紅字傳票） |
| reversed（已反沖）| 不可動 |

**編輯 pending 傳票**（v2.7.8+）：
- pending 行右側「編輯」按鈕 → 跳 modal
- 可改：日期、說明、全部分錄（科目 / 借 / 貸 / 摘要）
- 增 / 刪分錄、即時計算借貸合計、平衡才可儲存
- 已過帳的不再開放此按鈕，必須先「反沖」沖回再重建

**為什麼已過帳不能直接改**：會計慣例是「保留原始軌跡」—— 已過帳的錯誤要走反沖（產生紅字回沖傳票），不直接修改原單，否則無法滿足查帳要求。

### 電子發票（Phase 1：Turnkey 直連）

整體流程：國稅局核配字軌 → ADMIN 在「發票配號」頁輸入 → 「應收帳款」頁點「開立發票」→ 系統取號並產生 MIG 3.2.1 C0401 XML → 寫入 Turnkey 匯入目錄 → Turnkey 上傳國稅局 → 回執放 outbound → 跑 `sync-einvoice-status` CLI 反寫狀態。

**前置設定（⚙ ADMIN 一次性）**：
- 向國稅局申請「電子發票專用字軌核定通知書」並拿到配號區間
- 公司機房 / 伺服器安裝 Turnkey 程式，記下 inbound / outbound 目錄絕對路徑
- 到「公司資料」頁設定 `settings.einvoice.turnkeyInboundDir` / `turnkeyOutboundDir`（目前需 SQL 直接寫 Tenant.settings）

**發票配號頁（⚙ ADMIN）**：
- 每兩個月新增一期：期別（如 `11311`）、字軌兩碼（如 `AB`）、起訖號（如 `00000000` 至 `99999999`）
- 系統依 FIFO 取號；某批配號用完會自動停用
- 可手動停用（如作廢整批）或重新啟用

**開立**：
- 到「應收帳款」列表 → 找到要開票的單 → 點「開立發票」
- modal 會帶入客戶資料與銷貨品項，可微調
- 買受人型態：B2B（8 碼統編）／ B2C（統編自動填 `0000000000`）
- 送出後系統取號 → C0401 XML 寫入 Turnkey 匯入目錄 → 狀態 `issued`
- 該 AR 的「發票號碼」欄位會自動回填
- 銷貨單 PDF 再次下載時自動顯示發票號碼與開立日

**作廢**（⚙ ADMIN）：
- 到「電子發票」列表點紅色「作廢」→ 輸入原因 → 產生 C0501 XML → 狀態 `voided`
- 該 AR 的「發票號碼」快取會自動清空
- 銷貨單 PDF 會標紅「已作廢」
- 作廢僅限當期（跨期需走折讓單，Phase 2）

**狀態同步**：
- `issued` — XML 已寫入 inbound，待 Turnkey 上傳
- `confirmed` — 國稅局回執 OK（`<invoiceNo>*.xml` 出現在 outbound 且非 REJECT）
- `rejected` — 回執檔名含 `REJECT`，原因留在 `rejectReason`
- `voided` — C0501 已送出

執行 `npx tsx src/tools/sync-einvoice-status.ts` 由 cron 或手動定期跑；回執檔處理完會改名為 `*.processed-<epoch>`。

**Phase 2 待做**：載具（手機條碼／自然人憑證）、捐贈碼、折讓單（D0401/D0501）、自動上傳排程、LINE/LIFF 直接開立。

**證明聯 PDF 格式**（v2.4.0+）：
- **B2C**（無統編 / 載具 / 捐贈）：80mm 熱感紙樣式，含 Code39 條碼與雙 QR Code（AES-128 加密驗證）
- **B2B**（買方有 8 碼統編）：A5 直式格式，含買方框、應稅／零稅率／免稅勾選列、中文大寫金額、賣方框 + 蓋章區（自動套用發票章）
- 系統依買方統編自動分派；下載連結同一個 `/proof.pdf`

### 發票章

公司資料頁底部可上傳發票章 PNG（≤ 2MB，建議透明背景方形）。上傳後自動套用至：

- **報價單 PDF**：右下角總計表下方蓋章
- **B2B 電子發票證明聯**：右下「營業人蓋統一發票專用章」框內

可調整透明度（0.1–1，預設 0.85）。檔案存於 Fly volume `/data/stamps/<tenantId>.png`，每位 ADMIN 都可重新上傳。

## 常見問題

### Q: LINE 訊息沒回應？
1. 檢查是否加 Bot 為好友（封鎖狀態不會收到）
2. Bot 最近是否有被訊息淹沒（30 秒內大量訊息會 rate limit）
3. ADMIN 到「異常紀錄」看有沒有錯誤

### Q: PDF 連結過期？
- 短連結預設 7 天 TTL
- 過期後請到後台列表重新點「下載 PDF」→ 會產生新連結

### Q: 名片 OCR 抓不到資料？
- 檢查照片：文字清晰、光線足夠、避免反光
- 若 `GOOGLE_VISION_API_KEY` 未設，系統會直接回「請用文字表單」

### Q: 後台登入失敗？
- 員工編號大小寫必須完全相符
- 多公司員工請填公司名稱關鍵字
- 密碼錯 5 次不會鎖帳號（但 IP 會記在 ErrorLog）

### Q: 銷貨單 PDF 內勤電話/地址空白？
- 因為過去建的單沒存業務員資料 → PDF 會自動 fallback 用 `Employee.phone/address` 補
- 新建的單會直接帶入，無此問題

### Q: Prisma schema 改了，DB 還沒跟上怎麼辦？
- ADMIN 請到 Supabase SQL Editor 手動跑 ALTER TABLE
- 程式部分 route 有 P2022 fallback（schema drift 保護），短暫不一致不會讓整個服務掛掉

### Q: 每日自動備份沒收到？
- 先確認 `.env` 有設 `BACKUP_EMAIL_TO`、`SMTP_*`
- ADMIN 到「異常紀錄」篩 `source=job.daily-backup`
- 手動驗證：
  ```
  POST /api/statements/backup
  Cookie: ep_session=<你的 session>
  ```

### Q: 問題沒列在這？
請聯絡系統管理員（側欄底部顯示的版本資訊會幫助除錯）。

## 工作日誌（v2.8.0+）

業務員每次拜訪客戶留一筆紀錄。LINE chat 與後台都能填。

### 從 LINE 填
1. 在 LINE 主聊天輸入「日誌」，或從 rich menu「管理 → 工作日誌」進入
2. 機器人依序問：
   - 拜訪日期（`2026-05-12` / `今天` / `昨天`）
   - 客戶關鍵字 → 機器人模糊搜尋，多筆結果用 Flex carousel 讓你選一張
   - 拜訪內容（自由文字）
   - 下次行動日（`2026-05-20` 或 `無` / `跳過`）
3. 顯示摘要 → 按「送出」建立

### 從後台填
- 側欄「工作日誌」→「+ 新增日誌」
- 客戶欄是 autocomplete（從現有客戶選），不能填一個不存在的公司

### 權限
- SALES：只看自己建立的紀錄
- ADMIN / ACCOUNTING / VIEWER：看全租戶；VIEWER 無法寫入

## 客戶結帳資訊（v2.8.0+）

客戶 modal 加 4 欄位，AR 建單時自動依這些欄位算到期日：

| 欄位 | 用途 |
|---|---|
| 結帳日 (1-31) | 訂單日 ≤ 結帳日 → 計入本月帳；否則計入下月 |
| 固定付款日 (1-31) | 結帳月 + 帳期月數後，這天為到期日（若 > 該月最後一日自動 clamp） |
| 付款方式 | 開票 / 現金 / 轉帳 |
| 建立業務 | 員工選單；SALES 看自己建的客戶用這個 |

### dueDate 新公式

```
stmtDay = customer.statementDay ?? 31  # 月底
if orderDate.day ≤ stmtDay → 本月結；否則下月結
monthsAdd = floor((paymentDays ?? 30) / 30)
targetMonth = billingMonth + monthsAdd
if fixedPaymentDay 有設 → dueDate = (targetYear, targetMonth, min(fixedPaymentDay, 該月最後一日))
else                   → dueDate = endOfMonth(targetMonth)
```

範例：
- 訂單 2026-05-08，結帳日 10、固定付款日 20、paymentDays 30 → dueDate 2026-06-20
- 訂單 2026-05-12，結帳日 10、固定付款日 20、paymentDays 30 → 下月結，dueDate 2026-07-20
- 客戶 4 欄位都不設 → 沿用舊月底公式

## 月結帳單批次下載（v2.9.0+）

帳款頁有兩個按鈕：

- **📄 產生月結請款單**：選一個客戶 + 年/月 → 開單張 PDF（原功能）
- **📦 批次月結帳單（ZIP）**：選年/月 → modal 列出該月有未付的客戶 + 未付金額 + 最早到期日 → 「下載全部 ZIP」一鍵抓所有 PDF

ZIP 內檔名格式：`{客戶名}-YYYYMM.pdf`。

「該月有未付」定義：AR 的 `billingMonth` 等於指定月、且 `isPaid=false`。

## 名片 OCR：LINE 上傳避免壓縮（v2.8.0 確認）

LINE 拍照走 `image` 訊息會被 LINE 壓縮、辨識率下降。解法：

- **手機**：LINE 對話內按「＋」→「檔案」→ 從相簿選原圖（會以 `file` 訊息送出，保留原檔）
- **電腦**：後台「客戶」頁工具列「📇 名片辨識上傳」可上傳本地檔
- **副檔名須是** `.jpg` `.jpeg` `.png` `.heic` `.webp` `.gif`

機器人收到後跑 Google Vision OCR、抽公司/聯絡人/電話/Email/統編/地址 → 顯示三個選項：

- **✅ 直接建立**：OCR 結果直接建檔（v2.7.x 行為）
- **✏️ 編輯後建立**（v2.9.0+）：進入逐欄問流程，每欄顯示 OCR 結果，使用者可：
  - 回「OK」保留 OCR 值
  - 直接輸入新值覆蓋
  - 回「跳過」清空該欄
  - 回「取消」結束流程
  - 走完 6 欄後顯示摘要 → 按「建立」即新增
- **❌ 取消**

當辨識結果有空欄、錯字或要補資料時，用「編輯後建立」最方便。

## 產品備註匯入（v2.9.0+）

EK 系列 14 個產品的「產品特點與摘要」與「對應產品（國外牌號）」已內建在 `src/tools/import-product-notes.ts`。

### 本機跑

```bash
# Dry-run（只看會改什麼，不寫）
npx tsx src/tools/import-product-notes.ts <tenantId>
# 或：npm run import:product-notes -- <tenantId>

# 實際寫入
npx tsx src/tools/import-product-notes.ts <tenantId> --confirm
```

### 寫入規則

- 原 note 為空 → 直接寫產品特點 + 對應產品（若有）
- 原 note 有內容（例如「3/30 進價漲 1500」）→ **在後面換行追加** 產品特點，不刪原內容
- 若 note 已包含相同產品特點 → 跳過（冪等，可重複跑）

### 匹配規則

依產品 **name** 而不是 `code`。DB 中 `Product.code` 是 `MP-NNN` 序號，`Product.name` 才是 `EK-SS-6280 1/200` 這種規格代號。同一 PDF 條目通常對應到 1/200 + 1/19 兩個容量規格。
